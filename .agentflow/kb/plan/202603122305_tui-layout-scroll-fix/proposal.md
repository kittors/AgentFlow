# Proposal: TUI 布局与滚动修复

## 目标

- 修复主菜单在小终端和窄终端中的布局溢出问题。
- 让 `CLI 状态`、可执行文件路径和交互动作结果都留在 TUI 内部显示，不再泄漏到外层终端缓冲区。
- 接管鼠标滚轮事件，避免在 alt-screen 中把滚动交给终端外层 scrollback。

## 方案

- 将 `internal/ui` 的选择器从“多张高卡片纵向堆叠”改为“紧凑列表 + 详情面板”。
- 主菜单右侧/下方详情面板统一显示：
  - 当前动作说明
  - 环境状态（可执行文件路径、CLI 状态、更新提示）
  - 最近一次交互动作结果
- 在 Bubble Tea 程序启动时启用 `WithMouseCellMotion()`，并将滚轮映射为列表上下切换。
- 对视图总高度和详情区域内容做裁剪，保证渲染结果不超过终端可用区域。
- 对交互式 `install` / `uninstall` / `update` / `clean` 增加 panel 化返回值，避免再次直接向外层 stdout 打印。

## 涉及文件

- `internal/ui/ui.go`
- `internal/ui/ui_test.go`
- `internal/app/app.go`

## 风险

- 极小终端下仍然会有内容截断，但应该表现为 TUI 内部省略，而不是整体溢出到终端外部。
- 交互式更新仍然只能提示“重启进入新版本”，因为当前进程不会热切换到新二进制。

## 验收

- 主菜单中可见内嵌状态面板，不再出现 `CLI 状态` 输出滞留在 TUI 外部。
- 鼠标滚轮在菜单中切换选项，不再触发外层终端滚动。
- `go test ./...` 通过。
- 交互式启动首屏与主菜单均能在受限尺寸下渲染完成.
